# 字幕翻译执行规范

## 适用范围

适用于影视字幕翻译任务，尤其是电影、剧集、纪录片等带时间轴字幕文件的翻译工作。

## 当前生效规则

1. **如果你提供了 IMDb 编号**  
   在正式翻译前，先查看对应 IMDb 页面并查询 Wikipedia 等相关资料站点，了解影片或剧集的类型、背景、人物关系与整体风格，以便让译文更贴合作品本身，而不是机械直译。

2. **译文口吻要贴合作品类型、背景设定和人物语气**  
   翻译前应结合影片/剧集背景、类型、年代、主要设定与人物关系来确定整体中文口吻，避免只按字面逐句硬译。  
   - 喜剧、科幻喜剧、动作喜剧等作品，要保留台词的轻松、反应式幽默、误会感和人物节奏；口语台词应自然、有中文对白感，不因过度直译而失去风趣。  
   - 犯罪、惊悚、纪录片、奇幻、历史剧等类型，应分别采用符合题材的语气：该紧张时紧张，该冷静时冷静，该正式时正式。  
   - 涉及特定年代或经典老片时，译文可适当保留时代感，但不要故作文绉绉，也不要过度网络化。  
   - 涉及地域语言、俚语、脏话、讽刺、双关、口头禅时，应优先传达语气和功能，而不是逐字照搬；必要时用更贴近中文观众理解的表达替代。  
   - 涉及科技、生物、医疗、军事、法律等专业内容时，术语要准确；在不牺牲准确性的前提下，让台词仍像角色自然说出口。

3. **输出结果以可直接使用的字幕格式为准**  
   默认输出和原字幕文件相同的文件格式（如 srt），确保拿去继续剪辑、播放或校对时不需要大改。

4. **严格按照原字幕时间轴逐句翻译**  
   不擅自改动时间码、编号和基本结构。

5. **为避免时间轴错乱，不使用 Python 进行批处理**  
   后续类似任务中，采用更稳的逐段处理方式，不走整文件自动批量替换路线。

6. **每段翻译完成后，立即核对该段时间轴**  
   不是整份翻完再统一检查，而是边翻边核对，确认这一段无误后再继续下一段。

7. **整份字幕翻译完成后，再与原始字幕整体校对一次时间轴**  
   重点检查是否存在：
   - 只有原文、没有译文
   - 只有译文、没有原文
   - 原文与译文字数长度比异常
   - 段落错位、漏译、串段

8. **完成翻译后，增加“语境与表达专项检查”**  
   在整体时间轴校对之外，还应专项检查是否存在以下问题，并在需要人工判断时先列表标注，而不是直接强行改动：
   - 译文明显脱离前后文，单句看似通顺，但放回剧情、动作或人物关系中不自然；
   - 英文双关、歌名、歌词、片名、广告语、典故、固定表达、俚语或口头禅被按字面直译，导致原本的幽默、暗示或文化功能丢失；
   - 人物口吻不顺，例如角色本应慌张、调侃、强硬、讽刺或口语化，但译文过于书面、机械或不符合人物状态；
   - 科技、生物、医疗、军事、法律等专业词在上下文中可能不准确，或虽为字典释义但不符合该场景的专业用法；
   - 专有名词、片名、人名、绰号、歌曲名等可能需要统一译法或加引号区分语境。

   对这类条目，优先以表格列出：序号、时间轴、英文原文、当前译文、需要判定的原因、可考虑方向，由用户确认后再定点修改字幕文件。

9. **跨集翻译时，专有称谓必须保持一致（单集翻译可忽略）**  
   包括但不限于：
   - 人名
   - 地名
   - 机构名
   - 商品名
   - 组织名
   - 固定称呼
   - 口头禅或重复性术语

10. **遇到跨集项目时，先建立并持续维护“专有称谓参考词汇表”（单集翻译可忽略）**  
    后续各集翻译以该表为准，避免同一对象出现多种中文译法。  
    输出 csv 文件，栏目如下：
    - category
    - source_term
    - translated_term
    - variant_or_alias
    - translation_basis
    - first_seen_episode
    - first_seen_timestamp
    - source_evidence
    - notes

11. **如已有官方双语字幕或既有译本可参考（单集翻译可忽略）**  
    优先提取其中已经稳定使用的人名、地名、机构名等译法，作为后续统一标准。
